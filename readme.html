Projectbeschrijving

Kleire.be is de persoonlijke e-commerce website voor keramiek-kunstenares Claire Bosmans (“Klei’ re”). Het hoofddoel is het tonen en rechtstreeks verkopen van handgemaakte keramische objecten (vaasjes, kopjes, sculpturen, enz.) aan consumenten. Doordat Claire zelf weinig technische ervaring heeft, is de site bewust zo simpel mogelijk opgezet:

  • Volledige WooCommerce-integratie (producten, winkelwagen, afrekenen, bestellingen).
  • Gebruiksvriendelijke pagina’s (mobielvriendelijk, grote knoppen, duidelijke instructies).
  • Minimalistische backend (eenvoudige knoppen voor het toevoegen van nieuwe producten en beheren van bestellingen).
  • Geen overbodige franje: alleen datgene wat Claire nodig heeft, met maximaal gebruiksgemak.

Doelgroep & Belangrijkste Functionaliteiten

Doelgroep

  • Particuliere consumenten die handgemaakte keramiek (servies, vazen, sculpturen) willen kopen.
  • Familieleden of kennissen van de kunstenaar die snel een uniek cadeautje zoeken.

Belangrijkste Functionaliteiten

  1. Productbeheer (WooCommerce)
    • Eenvoudige productupload: afbeeldingen, titel, prijs, omschrijving, voorraad.
    • Automatische weergave op productpagina’s.
  2. Orderbeheer (Winkelmanager)
    • “Bestellingen Overzicht”-pagina (template) voor winkelmanager om orderstatus bij te werken.
    • Orderstatus wijzigen (In behandeling, Voltooid, Geannuleerd) in één klik.
  3. Formulieren & Contact
    • Ninja Forms voor contact- en vraagformulieren.
    • Eenvoudige e-mailnotificaties en opvolgingsmails.
  4. Beeldoptimalisatie & Performance
    • WP-Smush / Smush-WebP om afbeeldingen automatisch te comprimeren en WebP-versies te genereren.
    • IMSANITY om uploads automatisch in te krimpen naar geschikte afmetingen.
  5. Beveiliging & Privacy
    • Better-WP-Security (iThemes Security) voor basisbescherming tegen brute-force en exploits.
    • Complianz-GDPR voor cookiebanner, privacy policy en EU-GDPR-compliance.
    • WP-Hide-Login voor het verbergen van de standaard /wp-admin loginpagina.
  6. Backup & Update-beheer
    • UpdraftPlus voor automatische back-ups naar externe locaties (Dropbox, Google Drive).
    • Duplicator voor migraties en “export → import” van site + database.
    • Woo-Update-Manager specifiek voor WooCommerce-pluginupdates.
  7. Extra UX-tools
    • FileBird voor mappenstructuur in de mediabibliotheek.
    • MegaMenu voor eenvoudige menu’s in de header.
    • So-CSS voor kleine CSS-snippets via de customizer.
    • Loginizer om brute-force-pogingen op de inlogpagina te blokkeren.

Technische Stack

  • WordPress (versie 5.x of 6.x – exacte versie op productie)
  • PHP 8.0 (Nginx + PHP-FPM op de live-server)
  • MySQL / MariaDB (DirectAdmin-hosting)
  • Astra Theme (Parent)
  • Eigen Child-Theme “kleire-child-thema” (Zie sectie Child-Theme)
  • JavaScript & CSS:
    • jQuery (standaard bij WooCommerce)
    • Vanilla JS voor simpele event-handlers in custom page-templates
    • CSS (mobile-first, alleen overrides in child-theme)

Opmerking: Deze repo bevat niet de volledige wp-config.php (staat lokaal) en mediabestanden (wp-content/uploads/) – alleen core, plugins, thema’s en child-theme.

Child-Theme “kleire-child-thema”

Belangrijkste Aanpassingen

Dit child-theme is gebouwd op basis van het Astra-parent-theme. De aanpassingen zijn bewust minimaal, om de laadtijd laag te houden en eenvoud te bevorderen:

  • Enqueue Child-Theme Stylesheet
    add_action( 'wp_enqueue_scripts', 'klei_re_enqueue_styles' );
    function klei_re_enqueue_styles() {
      wp_enqueue_style( 'astra-child-style', get_stylesheet_uri(), array( 'astra-theme-css' ), '1.0.0' );
    }
    Hierdoor blijft alle styling van Astra beschikbaar, en kunnen we alleen extra CSS toevoegen in style.css.
  • Tagbeschrijving op Productpagina’s toevoegen
    add_action('woocommerce_product_additional_information', 'toon_tag_tekst_bijkomende_info');
    function toon_tag_tekst_bijkomende_info() {
      $product_id = get_the_ID();
      $tags = get_the_terms($product_id, 'product_tag');

      if ($tags && !is_wp_error($tags)) {
        foreach ($tags as $tag) {
          $tag_desc = term_description($tag);
          if (!empty($tag_desc)) {
            echo '<strong>Tag:</strong> ' . esc_html($tag->name) . '<br>';
            echo nl2br(strip_tags($tag_desc, '<br>'));
          }
        }
      }
    }
    Dit voegt onder “Additional Information” op een productpagina de tags en hun beschrijving toe, waardoor klanten snel extra details vinden.

Page Templates

1. Bestellingen Overzicht (klantenpaneel.php)

Template Name:

/*
  Template Name: Bestellingen Overzicht (Winkelmanager)
*/

Doel: Alleen toegankelijk voor ingelogde gebruikers met WooCommerce-rechten (Shop Manager / Administrator). Toont een overzichtstabel van orders met kolommen voor order-ID, klantnaam, totaalbedrag en huidige status. Elke rij bevat een dropdown-knop of actieknoppen om de orderstatus te wijzigen (‘In behandeling’, ‘Voltooid’, ‘Geannuleerd’).

Gebruik:

  1. Maak in de WordPress-backend een nieuwe pagina aan, bijv. “Bestellingen Overzicht”.
  2. Kies bij “Page Attributes → Template” voor “Bestellingen Overzicht (Winkelmanager)”.
  3. Publiceer de pagina. Wanneer je inlogt als winkelmanager of admin, navigeer je naar deze pagina om orders snel te beheren.

2. Product Upload Form (producten.php)

Template Name:

/*
  Template Name: Product Upload Form (Winkelmanager)
*/

Doel: Eenvoudig frontend-formulier waarmee een winkelmanager (of Administrator) zonder de WooCommerce-backend in kan loggen en direct via de site een nieuw product kan toevoegen.

Functionaliteit:

  • Controle op is_user_logged_in() & current_user_can('manage_woocommerce'). Zo niet, redirect naar homepage.
  • Formuliervelden:
    • Producttitel (tekst)
    • SKU (tekst)
    • Prijs (nummer)
    • Omschrijving (textarea)
    • Selecteer hoofdcategorie + tags
    • Upload hoofdfoto + galerijafbeeldingen (meerdere bestanden)
  • Na versturen: sanitizen van input, aanmaken van WooCommerce-product met wp_insert_post(), update_post_meta() voor prijs/SKU, en wp_set_object_terms() voor categorieën/tags.

Gebruik:

  1. Maak in de backend een nieuwe pagina aan, bijv. “Product Upload”.
  2. Kies bij “Page Attributes → Template” voor “Product Upload Form (Winkelmanager)”.
  3. Publiceer. Log in als winkelmanager / admin, ga naar deze pagina om via formulier nieuwe producten toe te voegen.

Responsive & Eenvoudige UX

Mobile-First Design

  • Grote knoppen, leesbare lettergroottes en voldoende contrast.
  • Eenvoudige navigatie met hamburger-menu en 1-koloms layout.
  • Product- en bestelpagina’s passen automatisch aan smartphonebreedte.

Desktop UX

  • Brede productgalerijen, duidelijke call-to-action (“In winkelwagen”, “Checkout”).
  • Checkout-pagina beperkt tot strikt noodzakelijke velden: naam, adres, e-mail, betaalmethode.

Admin UX (Backend Vereenvoudigd)

  • Alleen menu-items die Claire nodig heeft:
    • Producten → Nieuwe product (of via frontend form “Product Upload”)
    • WooCommerce → Bestellingen (alternatief via “Bestellingen Overzicht” frontend-pagina)
    • Media → Bibliotheek (met FileBird mappenstructuur)
    • Ninja Forms → Formulieren
    • Instellingen → Algemeen, Permalinks, Betalingen
  • Overbodige menu-items (bv. “Appearance → Widgets/Themes”) zijn verborgen via White Label CMS.

Installatie & Opzetten (Voor Interne Demonstratie)

Let op: Deze handleiding is bedoeld voor collega’s of toekomstige werkgevers om inzicht te krijgen in hoe de site lokaal op te zetten. Dit is niet de officiële klant‐deploy (mediabestanden, licenties en live‐wp-config ontbreken).

  1. Repo Klonen
    https://github.com/evy-houbrechts/kleire.be.git
  2. Nieuwe Database Aanmaken
    Gebruik phpMyAdmin, MySQL CLI of een andere client. Maak een lege database aan (bijv. kleire_local) en noteer gebruikersnaam/wachtwoord.
  3. Kopieer & Configureer wp-config.php
    cp wp-config-sample.php wp-config.php
    Open wp-config.php en vul in:
    define('DB_NAME', 'kleire_local');
    define('DB_USER', 'db_gebruiker');
    define('DB_PASSWORD', 'db_wachtwoord');
    define('DB_HOST', 'localhost');
    define('WP_DEBUG', true); <!-- Voor ontwikkelmodus -->
  4. Plugins & Thema’s Controleren
    • Alle plugin-folders staan al in wp-content/plugins/. Activeer in WP-backend: WooCommerce, Ninja Forms, Better-WP-Security, Complianz-GDPR, etc.
    • Ga naar Weergave → Thema’s, activeer eerst Astra, daarna Klei’ re Child.
  5. Composer & npm (optioneel)
    Deze repo bevat geen composer.json voor WordPress-core. Mocht je in toekomst dependencies toevoegen, gebruik dan composer install.
    Voor het child-theme zijn geen npm-scripts vereist. Alle CSS-overrides zitten in style.css.
  6. Maak wp-content/uploads Map aan
    • Maak in wp-content/ de map uploads/ aan en geef schrijfrechten.
    • Upload demo-media via WP-backend (Media → Nieuw bestand).
  7. Start de Lokale Webserver
    • Gebruik Local by Flywheel, XAMPP, MAMP of Laragon. Zorg dat de DocumentRoot wijst naar de root van deze repo (/path/to/kleire.be).
    • Open in je browser: http://localhost/kleire.be/. De WordPress-installatie-wizard verschijnt.
  8. Permalinks & WooCommerce Instellingen
    Nadat installatie voltooid is, log in op http://localhost/kleire.be/wp-admin.
    • Ga naar Instellingen → Permalinks: kies “Berichtnaam” én klik “Wijzigingen opslaan”.
    • Ga naar WooCommerce → Instellingen: configureer land (België), valuta (EUR), betaalmethoden (bijv. PayPal sandbox) en verzendopties (vaste prijs, gratis verzending, etc.).
  9. Pagina’s Aanmaken
    • Bestellingen Overzicht – Nieuwe pagina, kies template “Bestellingen Overzicht (Winkelmanager)”.
    • Product Upload – Nieuwe pagina, kies template “Product Upload Form (Winkelmanager)”.
    Publiceer beide pagina’s.
  10. Testen
    • Front-end: Controleer homepage, productpagina’s (tags + beschrijving), winkelwagen en checkout.
    • Winkelmanager: Log in als admin, ga naar “Bestellingen Overzicht” om orderstatus te wijzigen. Ga naar “Product Upload” om via formulier nieuwe producten toe te voegen.
